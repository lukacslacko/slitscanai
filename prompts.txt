# Plan: Rewrite Panorama Generation in main.py

## Context

The current `generate_panorama` method (lines 599-885) uses per-frame SIFT matching with an expanding search window and per-frame affine rotation to track the tram and extract slices. This approach is fragile (crashes on empty ROI slices) and overly complex. The stabilization step works well. We replace everything after it with a simpler, more robust algorithm based on phase correlation and a single global rotation.

## Algorithm Summary

1. User selects a frame (slider) and tram region (ROI) on the stabilized view
2. Estimate tram movement direction from ~10 frame pairs near the selected frame
3. Rotate ALL stabilized frames once by that angle to make movement purely horizontal
4. For each frame pair, estimate per-frame horizontal displacement using phase correlation on a tracked ROI
5. Extract full-height vertical slices at the ROI center, concatenate into panorama

### 3. Replace `generate_panorama` method (lines 599-885)

The new method has 5 phases:

**Phase 1 - Direction estimation:** Use `cv2.phaseCorrelate` on the tram ROI across ~10 frame pairs centered on `tram_frame_idx`. Apply Hanning window. Take median of dx/dy samples. Compute `angle = atan2(median_dy, median_dx)`.

**Phase 2 - Angle validation:** If `abs(angle_deg) > 20` (and not near 180), warn and abort. Tram movement should be near-horizontal.

**Phase 3 - Rotate all frames:** Apply `cv2.getRotationMatrix2D(roi_center, angle_deg, 1.0)` + `warpAffine` to every stabilized frame. Use `BORDER_REPLICATE` to avoid black edges. Store rotated frames in a temporary list.

**Phase 4 - Per-frame displacement:** For each consecutive frame pair in the rotated frames:
- Track the tram's x-position (starts at `roi_cx`, accumulates dx each frame)
- Extract the ROI-height horizontal band at the tracked position (with padding for search range)
- `cv2.phaseCorrelate` on these bands to get per-frame dx
- Fallback to rolling median if confidence is low

Key detail: we use a **tracked ROI region** (not full-frame-width band) for phase correlation because the stabilized background is static — full-width phase correlation would return dx≈0 (background dominates). By tracking the ROI to where the tram currently is, phase correlation measures the tram's actual displacement.

**Phase 5 - Slice extraction:** For each frame:
- `slice_width = max(1, round(abs(dx)))`
- Cut a **full-height** vertical strip centered at `roi_cx`, width = `slice_width`
- Concatenate all strips, reverse if tram moved right (so nose appears on the left)
